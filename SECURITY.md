# Security Policy

## Supported versions

Security fixes are provided for the latest version on the `master` branch and
the latest published release.

## Reporting a vulnerability

Please do not open a public issue for a suspected security vulnerability.

Use GitHub's **Report a vulnerability** feature in this repository's Security
tab to send a private report. Include a clear description, affected version or
commit, reproduction steps, impact, and any suggested mitigation.

We will acknowledge the report, investigate it privately, and coordinate a fix
and disclosure timeline with you when appropriate.
